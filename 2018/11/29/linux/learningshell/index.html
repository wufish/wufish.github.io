基础

  • Bash（Bourne Again Shell），是大多数GNU/Linux默认的shell环境

  • 命令提示符

  • shell 脚本通常以 shebang起始

  • 脚本运行方式：

    bash script.sh
    chmod a+x script.sh
    ./script.sh

  • 启动shell之前会执行一组命令来定义如提示文本、颜色等设置，这组命令位于~/.bashrc（对于登陆shell则是~/.bash_profile）（登陆shell是登陆主机后获取的shell，如果是图形界面环境打开了一个shell则不是登陆shell）

  • 历史命令记录文件，~/.bash_history

  • 命令定界符;

当shell是交互式登录shell时，读取.bash_profile文件，如在系统启动、远程登录或使用su -切换用户时；当shell是交互式登录和非登录shell时都会读取.bashrc文件，如：在图形界面中打开新终端或使用su切换用户时，均属于非登录shell的情况。简单的说，.bash_profile只在会话开始时被读取一次，而.bashrc则每次打开新的终端时，都会被读取。

.bash_logout在退出shell时被读取。

终端打印

  • 命令注释以 # 开始知道行位

  • echo 打印特殊字符，如果放入单引号中需要使用转义字符或者放入双引号中

  • 变量替换在单引号中无效

  • printf格式化字符串,%s、%c、%d、%f

变量

  • bash中每一个变量的值都是字符串。无论你给变量赋值时有没有使用引号，值都会以字符串的形式存储。

  • var1 = value 和 var1=value 前者是相等操作，后者才是赋值操作（变量名和值之间不能有空格）,value如果有空格需要用引号包裹起来

  • $var1 或者${var1} 引用变量var1

  • export 设置环境变量

  • 使用单引号，变量不会被扩展，如果未定义变量也是直接打印

  • length=${#var},获取变量值的长度

  • 环境变量：

$SHELL 或者 $0 获取当前使用的shell

UID == 0 是超级用户root

PS1， 设置提示文本。\u 用户名、\h 主机名、\w当前工作目录

  • 函数添加环境变量

    首先检查第二个变量是否存在，eval 两次扫描变量，执行复杂的命令

  • shell 参数扩展：

    ${parameter:+expression} 如果parameter有值且不为空，则使用expresion的值

数学运算

整数

  • let 让变量直接执行基本的算术操作，变量前不用加$；[] 和let类似：

浮点数运算

  • bc

echo “4 * 0.56” | bc
no=54;
result=echo "$no * 1.5" | bc

  • 设定小数精度

echo “scale=2;3/8” | bc

  • 进制转换

no=100
echo “obase=2;$no” | bc
1100100
no=1100100
echo “obase=10;ibase=2;$no” | bc
100

echo “sqrt(100)” | bc #Square root
echo “10^10” | bc #Square

文件描述符和重定向

  • 文件描述符：stdid、stdout、stderr，文件描述符是与某个打开的文件或数据流相关联的整数。分别是0,1,2

  • 重定向操作符：> 覆盖文件，>>追加文件。> 等同于 1>

  • cat a* 2> err.txt #stderr被重定向到err.txt

  • cmd 2>/dev/null #对stderr 重定向, /dev/null 是特殊的设备文件，他接受到的任何数据都会被丢弃

tee命令只接收到来自stdin的数据。它将stdout的一份副本写入文件out.txt，同时将另一份副本作为后续命令的stdin。命令cat -n将从stdin中接收到的每一行数据前加上行号并写入stdout

  • cmd < file 文件重定向到命令
  • exec 创建自己的文件描述符
